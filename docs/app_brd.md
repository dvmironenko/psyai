# Описание приложения PsyAI. Кабинет психолога

## Описание пользовательской функциональности

### Детали бизнес‑логики

- **Регистрация** – пользователь вводит email, пароль (минимум 8 символов) и имя. После регистрации ему отправляется письмо с подтверждением email.
- **Аутентификация** – вход по email и паролю. При успешной проверке выдаётся JWT‑токен сроком 30 минут.
- **Деактивация аккаунта** – пользователь может деактивировать свой профиль, после чего `is_active` становится `false`. Деактивированный пользователь больше не может входить в систему, но данные сохраняются для статистики.
- **Планы доступа** – каждый пользователь имеет поле `plan` (0‑free, 1‑basic, 2‑premium, 3‑ultra). План определяет доступ к функциям: например, только пользователи `premium` и выше могут создавать собственные сессии.
- **Администрирование** – администраторы (`is_admin=true`) могут просматривать, редактировать и деактивировать любые аккаунты. У них есть доступ к статистике использования.
- **Восстановление пароля** – пользователь запрашивает ссылку на сброс, получающуюся по email. Ссылка действительна 15 минут.
- **Сессии** – при каждом входе генерируется новый JWT‑токен, который хранится в cookie с флагом `HttpOnly` и `Secure`. Токен обновляется автоматически при каждом запросе.
- **Безопасность** – все входные данные валидируются сервером, используется `bcrypt` для хеширования паролей, защита от CSRF через Flask‑Session.
- **Логи** – все ошибки аутентификации записываются в лог с уровнем `warning`.


Пользователь приложения может:

- Зарегистрироваться в приложении
- Войти в систему
- Изменить данные о себе
  - Деактивация аккаунта (установка поля `is_active` в `false`)
- Выйти из системы
- Восстановить забытый пароль

## Права пользователей

Вместо удаления записи пользователя, система устанавливает значение поля `is_active` в `false`, что делает пользователя неактивным, но сохраняет его данные в базе данных.

## Определения пользователей и уровни доступа

- **Зарегистрированный пользователь** — пользователь, который прошёл процесс регистрации и имеет аккаунт в базе данных.
- **Аутентифицированный пользователь** — пользователь, который прошёл процесс аутентификации и имеет действующий JWT токен для доступа к защищённым ресурсам.
- **Пользователь с уровнем доступа (PLAN)** — поле `plan` в таблице пользователей имеет тип integer и принимает значения:
  - `0` – free
  - `1` – basic
  - `2` – premium
  - `3` – ultra
 Уровень влияет на доступ к определённым эндпоинтам и функциональности.

## Доступ к страницам

- Для всех пользователей (зарегистрированных и незарегистрированных) доступна главная страница по пути `/`. С этой страницы пользователи могут авторизоваться и зарегистрироваться.
- Все остальные страницы доступны только авторизованным пользователям с разрешенным уровнем доступа и администраторам.
- Администраторы — это пользователи, у которых поле `is_admin` равно `true`.

## Основные сценарии использования

1. **Регистрация пользователя**
   - Пользователь может зарегистрироваться в приложении
   - Система создает новую запись пользователя в базе данных

2. **Аутентификация пользователя**
   - Пользователь может войти в систему с использованием логина и пароля
   - Система проверяет учетные данные и выдает JWT токен

3. **Управление профилем пользователя**
   - Аутентифицированный пользователь может изменить свои данные
   - Пользователь может деактивировать свой аккаунт (установка `is_active = false`)

4. **Доступ к защищенным ресурсам**
   - Аутентифицированные пользователи имеют доступ к dashboard
   - Администраторы имеют доступ к админ-панели с расширенными возможностями

5. **Административные функции**
   - Администраторы могут выполнять CRUD операции с пользователями
   - Доступ к административным эндпоинтам явно проверяет поле `is_admin` на каждом запросе

6. **Выход из системы**
   - Аутентифицированный пользователь может завершить сеанс работы
   - Система удаляет JWT токен из сессии

7. **Восстановление пароля**
   - Пользователь может запросить восстановление забытого пароля
   - Система отправляет инструкции на электронную почту пользователя
